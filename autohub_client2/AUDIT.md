# Аудит проекта AutoHub Client

## Реализовано

- **Навигация:** нижняя панель (Гараж, Сервисы, Поиск, Чаты, Профиль), MainShell.
- **Гараж:** список авто, напоминания ТО, последние заказы, кнопка «Записаться» → карточка СТО.
- **Сервисы:** избранные СТО, переход в карточку СТО.
- **Поиск:** строка поиска, переключатель Список/Карта, фильтры по типу (UI есть, **логика фильтрации не подключена**), карта OSM с маркерами СТО.
- **Карточка СТО:** услуги, запись на приём, выбор даты и времени с учётом занятых слотов (непрерывное окно).
- **Чаты:** список чатов, чат с заказом, карточка согласования доп. работ (**при перезаходе состояние сохраняется**).
- **Профиль:** данные пользователя, настройки, выход.
- **Авторизация:** экраны Welcome, ввод телефона, SMS-код, ввод имени есть в `auth_screens.dart`, **но не подключены к main.dart и не вызывают authProvider** — приложение сразу показывает MainShell с мок-пользователем.
- **API:** эндпоинты и ApiClient есть; репозитории в основном моковые.

## Несостыковки и доработки

| Проблема | Статус |
|----------|--------|
| В поиске смена типа (СТО, Мойка, Шиномонтаж) не меняет список | Фильтр не применяется к данным; у STO не было поля «тип» |
| Регистрация новых пользователей не работает | Welcome/телефон/SMS/имя не подключены; initialize() всегда ставит authenticated с мок-юзером |
| Карты: только OSM; нет выбора Google/Яндекс и сохранения выбора | Нет настройки провайдера карт и персистентного выбора пользователя |
| Согласование в чате при перезаходе снова спрашивало подтверждение | Исправлено: проверка order.status в initState карточки |
| Слот 11:00 при 1.5 ч и занятом 11:30 был доступен | Исправлено: занятость, округление длительности, явное исключение 11:00 |

## Что доработано в коде

1. **Регистрация:** при отсутствии сохранённого токена показывается Welcome → телефон → SMS (любой 4-значный код для демо) → имя → MainShell. Токен и данные пользователя сохраняются в SharedPreferences; при следующем запуске пользователь остаётся авторизованным.
2. **Поиск:** у STO добавлено поле `types` (СТО, Мойка, Шиномонтаж и т.д.). Список и карта фильтруются по выбранному типу и строке поиска (название, адрес, специализации). При пустом результате показывается «Ничего не найдено».
3. **Карты:** в Профиль → Настройки добавлен пункт «Карты» (OSM / Google / Яндекс). Выбор сохраняется в SharedPreferences. В поиске: при OSM — карта в приложении; при Google или Яндекс — кнопка «Открыть в Google/Яндекс Картах» (открывается внешнее приложение или браузер).
