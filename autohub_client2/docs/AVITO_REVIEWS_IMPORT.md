# Импорт отзывов с Авито при регистрации

## Цель
Дать пользователю (клиенту или в будущем — партнёру/СТО) возможность перенести отзывы с Авито в приложение AutoHub: показывать их в профиле или привязать к карточке СТО/мастера.

## Варианты получения данных

### 1. Ручной ввод (реализуемо сразу)
- Пользователь вводит отзывы вручную: автор, оценка, дата, текст.
- Плюсы: не зависит от Авито, легально, надёжно.
- Минусы: трудозатратно при большом количестве отзывов.

### 2. Вставка ссылки на объявление/профиль Авито
- Пользователь вставляет URL страницы с отзывами (например `https://www.avito.ru/user/...` или страница объявления с отзывами).
- **Ограничения**: у Авито нет публичного API для получения отзывов. Парсинг страниц:
  - против правил использования Авито и может привести к блокировке;
  - хрупкий (любое изменение вёрстки ломает парсер);
  - не подходит для продакшена без договорённости с Авито.
- Реалистичный вариант: по ссылке мы только **сохраняем URL** как «источник отзывов», а сами отзывы пользователь добавляет вручную или мы показываем инструкцию «скопируйте отзывы с Авито и вставьте ниже».

### 3. Импорт из файла (средняя сложность)
- Пользователь загружает файл (CSV/JSON), сформированный вручную или экспортом из другого сервиса.
- Формат, например: `author,rating,date,text` или JSON-массив отзывов.
- Плюсы: можно массово переносить, без парсинга Авито.

### 4. Официальная интеграция с Авито (будущее)
- Если появится API или партнёрская программа Авито для переноса отзывов — подключить.

---

## Рекомендуемый первый шаг

1. **Модель данных** — «импортированный отзыв»:
   - `id`, `authorName`, `rating`, `date`, `text`, `source` (например `avito`), `userId` (или `stoId` при партнёрском сценарии).
2. **Экран «Импорт отзывов с Авито»** (после ввода имени или из профиля):
   - Краткий текст: «Вы можете перенести отзывы с Авито. Укажите ссылку на страницу с отзывами (для себя) или добавьте отзывы вручную».
   - Поле «Ссылка на Авито» (опционально) — сохраняем как источник.
   - Кнопка «Добавить отзыв вручную»: форма (автор, оценка 1–5, дата, текст).
   - Список уже добавленных отзывов с возможностью удалить.
3. **Хранение**:
   - Пока — локально (SharedPreferences/локальная БД) или в состоянии приложения; при появлении бэкенда — API «мои импортированные отзывы» с привязкой к `userId`.

## Где встроить в онбординг

- **Вариант A**: после `NameInputScreen` показать необязательный экран «Импорт отзывов с Авито» с кнопками «Добавить отзывы» и «Пропустить».
- **Вариант B**: не показывать при регистрации, а только в профиле: раздел «Отзывы с Авито» / «Импорт отзывов».

## Дальнейшие шаги

- Ввести модель `ImportedReview` и хранилище (провайдер + SharedPreferences или API).
- Реализовать экран импорта (ручной ввод + опционально ссылка).
- Подключить отображение импортированных отзывов: в профиле пользователя и/или на карточке СТО (когда будет привязка отзывов к организации).
