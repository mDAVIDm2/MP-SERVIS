# API: `business_kind`, `organization_kind` и справочники

Краткий контракт для клиентов (Client / Business) и панели.

## Коды вида точки (`business_kind`)

Сервер нормализует значение через `normalizeOrganizationBusinessKind` (см. `src/organizations/organization-business-kind.ts`).

Поддерживаемые коды: `sto`, `car_wash`, `detailing`, `car_audio`, `tire_service`, `body_shop`, `glass`, `tuning`, `ev_service`, `other`. Неизвестное значение приводится к `sto`.

## Профиль организации

- В ответах профиля/списка организаций поле **`business_kind`** — основной код вида точки.
- В чатах дублируется **`organization_kind`** (историческое имя); по смыслу совпадает с `business_kind`.

## Справочник услуг

- **`GET /catalog/services`** — глобальный каталог; опциональный query **`business_kind`** фильтрует позиции по `allowed_business_kinds`.
- **`GET /reference/service-catalog`** (JWT):
  - с **`organization_id`** или с организацией в токене — группировка с фильтром по `business_kind` этой организации;
  - **без** привязки к организации — опциональный query **`business_kind`**: фильтр по виду точки (как у каталога без org).

## Настройки организации

При сохранении услуг/позиций из единого справочника сервер проверяет, что id вида `svc_*` разрешены для `business_kind` организации (`OrganizationsService`).

## Заказы

- В JSON заказа для клиента передаются **`organization_business_kind`** (нормализованный код вида точки) и **`organization_scheduling_mode`** (`staff_based` | `bay_based`). Если организация в ответе не загружена, оба поля могут быть **`null`**.
- Клиентское приложение подтягивает регламентные работы в гараж **только** из завершённых заказов с `organization_business_kind` в множестве **`sto`**, **`tire_service`**, **`ev_service`**; при отсутствии кода записи из заказа не создаются.
- **`POST .../assign-master`**: поле **`bay_id`** допустимо только при **`bay_based`**; пост должен существовать в настройках слотов; при смене поста выполняется проверка пересечения по времени с другими активными заказами.

## Режимы записи

- **`staff_based`** — слоты по мастерам; **`bay_id`** в заказе не используется.
- **`bay_based`** — слоты по постам; при именованных постах **`bay_id`** обязателен/проверяется при создании и при назначении.
