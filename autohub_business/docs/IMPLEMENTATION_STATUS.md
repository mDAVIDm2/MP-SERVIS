# Статус реализации AutoHub Business

Сверка с промптами: `AUTOHUB_STO_PROMPT.md`, `AUTOHUB_BUSINESS_MASTER_PROMPT.md`.  
Детальные пробелы и упрощения: **GAPS_AND_SIMPLIFICATIONS.md**.

## Реализовано

### Роли и навигация
- **Owner**: Главная (дашборд), Календарь, Заказы, Чаты, Профиль. В профиле: Организация, Клиенты, Персонал, Настройки.
- **Admin**: Заказы, Календарь, Чаты, Профиль (без дашборда). Клиенты, Персонал, Настройки в профиле.
- **Master**: только «Мои задачи» и «Профиль». Без цен и телефонов в заказах. **Фото по заказу** и **Доп. работы** — реализованы (API + экраны).
- **Solo**: Заказы, Календарь, Чаты, Профиль. Без раздела «Персонал», без кнопки «Назначить мастера».

### Роутинг
- **go_router**: маршруты `/`, `/login`, `/app`; redirect по auth (не авторизован → `/login`); для не-Owner при tab=0 редирект на tab=1. Переход на главный экран после входа через `context.go('/app')`. Вкладки синхронизированы с путём `/app?tab=...`.

### Авторизация
- Вход по телефону и SMS-коду. Сначала запрос к реальному API (send-code, verify-code); при ошибке сети — демо-вход: `1111` — мастер, `2222` — владелец, `3333` — самозанятый, любой другой 4-значный — администратор.

### API и WebSocket
- **OrderRepository**: загрузка заказов с API при старте; при ошибке сети — демо-данные. Подписка на WebSocket (order_updated / order_created). Мутации возвращают `Result`; при ошибке API — откат state и SnackBar в UI.
- **Auth**: реальный API verify + при fallback демо; после входа подключение WebSocket с токеном; при logout — отключение WS.
- **ChatRepository**: загрузка чатов/сообщений с API; отправка сообщений и запросов согласования через API; оптимистичное обновление; при ошибке — SnackBar. Подписка на WebSocket (chat_message / message / new_message).
- **OrganizationRepository, StaffRepository, SettingsRepository**: загрузка и сохранение через API организаций; при ошибке или отсутствии orgId — fallback в SharedPreferences. Pull-to-refresh на экранах.

### Заказы
- Список: активные / история. **Фильтры** по статусу, дате (с/по), мастеру. Детали заказа, звонок клиенту.
- Смена статуса, назначение мастера, отмена: вызов API; при ошибке — откат state и SnackBar.
- **completeOrderItem**: PATCH `/orders/:id/items`, вызов из репозитория; тап по позиции в детали заказа и в «Мои задачи» для отметки выполненной. При ошибке API — SnackBar.
- Создание заказа из календаря (форма + API или локально при ошибке).

### Дашборд (только Owner)
- KPI: заказов сегодня, выручка сегодня, требуют внимания. **По сотрудникам (сегодня)**: заказы и выручка по каждому мастеру. Список заказов на сегодня с переходом в детали.

### Календарь
- Вид **День** и **Неделя** (полоска дней, выбор дня, список заказов). Выбор даты (стрелки + календарь). Создание заказа (FAB) с передачей выбранной даты.

### Чаты
- Список чатов и сообщения — загрузка с API; при ошибке — демо только если список пустой. Отправка сообщений и запросов согласования через API; оптимистичное обновление; при ошибке — SnackBar. Шаблоны сообщений (вставка из настроек). Прочитанность при открытии.

### Клиенты
- Список клиентов СТО (агрегация по заказам). Карточка: телефон, звонок, **внутренние заметки** (SharedPreferences), история заказов с переходом в заказ.

### Мои задачи (Master)
- Текущая и следующие заказы, назначенные на мастера. Чек-лист работ с тапом для отметки выполненной (через API). **Фото по заказу**: экран с загрузкой с API, добавление фото (image_picker + POST), отображение через авторизованный запрос. **Доп. работы**: экран с добавлением позиции через `addExtraWorkItem` (PATCH items), SnackBar при ошибке.

### Профиль и настройки
- **Организация**: загрузка/сохранение через GET/PATCH `/organizations/:id`; кэш в prefs; pull-to-refresh. Бэкенд: маппинг `working_hours` при обновлении.
- **Персонал**: загрузка/приглашение/обновление/деактивация через API `/organizations/:id/staff`; кэш в prefs; pull-to-refresh.
- **Настройки**: услуги и цены, марки, слоты, уведомления, шаблоны сообщений — загрузка и сохранение через GET/PATCH `/organizations/:id/settings`; кэш в prefs; pull-to-refresh на экране «Настройки».

### Платформы и тема
- Android, Web, Windows/macOS. На широком экране — боковое меню. Тёмная тема, акцент #FF6B00.

## Не реализовано / упрощено (кратко)

- **Оплата** — экранов/полей оплаты и привязки к заказу нет.
- **Push-уведомления** — регистрация токена и список уведомлений не используются (endpoints есть в ApiEndpoints).
- **Аналитика по периоду / средний чек** — только «сегодня»; выбора периода нет.
- **Детальные маршруты** (например `/app/orders/:id`) — переходы через `Navigator.push`.
- **Refresh token**, **GET /profile** при старте — не реализованы.
- **Ответ клиента на согласование** в чате — обновление статуса карточки локальное; синхронизация с клиентским приложением зависит от бэкенда.

Подробно: **GAPS_AND_SIMPLIFICATIONS.md**.
