## 2025-03-11 — Восстановление mini-card расписания и Detail Panel (регрессии после правок)

**Дата/время:** 2025-03-11

**Изменённые файлы:**
- `autohub_business/lib/features/schedule/presentation/screens/schedule_desktop_grid.dart`
- `autohub_business/lib/features/orders/presentation/widgets/order_detail_panel.dart`

- Статус в mini-card «прыгал» вверх вместо закрепления внизу.
- При переключении на другой заказ detail panel иногда показывал состав первого открытого заказа (stale state).
- Текст в mini-card разного размера, статус не в блоке по центру внизу.

**Что исправлено:**

1. **Detail Panel — сброс при смене заказа**  
   В `OrderDetailPanel` добавлен `didUpdateWidget`: при смене `orderId` обнуляются `_loadedFallbackItems`, `_loadedFallbackTotalKopecks`, `_loadedFallbackTotalMinutes` и при необходимости запускается загрузка fallback для нового заказа. 3. **Mini-card (full)**  
   Структура: номер и время → авто → «Назначен: X» (если есть) → «Клиент: X» (если есть) → `Expanded(SingleChildScrollView)` со списком работ (до 4 с кружками, выполненные — зелёная галочка и зачёркивание) → внизу **закреплённый** блок статуса по центру (`Center(Container(... status)))`). Единая типографика: `fontSize: 10` для основного текста, без белого/разнокалиберного текста.

**Минимальная суть правки:**
- `order_detail_panel.dart`: добавлен `didUpdateWidget` с очисткой fallback при смене `orderId`.
- `schedule_desktop_grid.dart`: в `_buildCompactContent` и `_buildFullContent` статус отображается в `Center(Container(...))` внизу; в full добавлены строки «Назначен» и «Клиент», список работ до 4 пунктов с кружками; единый fontSize 10.

**Как проверить руками:**
1. Открыть расписание, кликнуть по mini-card заказа A — справа состав A. Затем кликнуть по заказу B — справа сразу состав B, без остатков A.
2. Убедиться, что статус в mini-card всегда внизу карточки в блоке по центру и не «прыгает».
3. У заказа с несколькими работами в mini-card видны пункты со списком и кружками (выполненные — зелёная галочка и зачёркивание).

## 2025-03-11 — Новые заказы: состав из чата в state, клик «выполнено» в панели

**Дата/время:** 2025-03-11

**Изменённые файлы:**
- `autohub_business/lib/core/repositories/order_repository.dart`
- `autohub_business/lib/features/orders/presentation/widgets/order_detail_panel.dart`

**Что было сломано:**
- У новых заказов (API возвращает пустой `items`) позиции были видны только в открытой карточке заказа (fallback из чата), но без возможности отметить выполнение; во вкладке «Заказы» и в mini-card расписания состав не отображался.

**Что исправлено:**
1. В **order_repository** добавлен метод `setOrderItemsIfEmpty(orderId, items)`: подставляет переданный состав в заказ в state только если у этого заказа сейчас пустой `items`. Не перезаписывает уже заполненный состав.
2. В **OrderDetailPanel** в `_loadFallbackFromChatIfNeeded()` после загрузки позиций из сообщения согласования (чат): список `ApprovalItem` конвертируется в `OrderItem` (id, name, priceKopecks, estimatedMinutes, isCompleted: false) и вызывается `setOrderItemsIfEmpty(widget.orderId, orderItems)`. Заказ в state получает состав один раз при первом открытии панели.
3. Дальше везде используются уже `order.items`: во вкладке «Заказы», в mini-card расписания и в панели заказа — отображаются одни и те же позиции, по ним работает клик «отметить выполненной» (toggle).

**Как проверить руками:**
1. Взять новый заказ (без состава в API). Открыть его карточку (из расписания, из «Заказы» или из чата) — после загрузки в панели должен появиться состав, по клику позиции отмечаются выполненными (галочка, зачёркивание).
2. Закрыть карточку, открыть вкладку «Заказы» — у этого заказа должен отображаться состав.
3. В расписании в mini-card у этого заказа должны отображаться те же задачи.

## 2025-03-10 — Зависание при перетаскивании миникарточки (feedback infinite height)

**Дата/время:** 2025-03-10

**Изменённые файлы:**
- `autohub_business/lib/features/schedule/presentation/screens/schedule_desktop_grid.dart`

**Что изменено:**

1. **Draggable feedback — ограничение высоты**  
   У виджета `feedback` у `Draggable<Order>` была задана только ширина (`SizedBox(width: _kUnassignedColWidth - 16)`), высота оставалась неограниченной. Это приводило к ошибке «BoxConstraints forces an infinite height», каскаду «RenderBox was not laid out» и «Cannot hit test a render box with no size», из‑за чего приложение «висело» при перетаскивании. Для feedback заданы явные размеры: `height: feedbackHeight`, где `feedbackHeight = (slotSpan * _kRowHeight - 8).clamp(_kRowHeight - 8, 280.0)`. Контент обёрнут в `ClipRRect` для обрезки по скруглению.

**Как проверить:** Перетащить миникарточку заказа в расписании на слот мастера — перетаскивание проходит без зависания и без жёлтых полос overflow.

## 2025-03-10 — Повторный клик скрывает панель, overflow, время в карточке, курсор и перетаскивание колонок

**Дата/время:** 2025-03-10

**Изменённые файлы:**
- `autohub_business/lib/features/schedule/presentation/screens/schedule_desktop_grid.dart`
- `autohub_business/lib/features/orders/presentation/widgets/order_detail_panel.dart`

**Что изменено:**

1. **Повторный клик по открытой миникарточке** — при клике по заказу, который уже выбран (панель открыта), выбор сбрасывается и панель скрывается (`_onOrderTap`: если `orderId == current`, выставляется `null`).

2. **Overflow 5px** — в панели заказа контент обёрнут в `ConstrainedBox(maxWidth: kOrderDetailPanelWidth - 32)`. У миникарточки на 1 слот: уменьшены отступы (vertical: 2), шрифты (10/9), фиксированная высота контейнера `_kRowHeight - 10` и `clipBehavior: Clip.hardEdge` для обрезки по высоте.

3. **Переход на следующий день** — подсчёт «лишних» прокруток вниз перенесён на колёсико мыши: вертикальная область обёрнута в `Listener(onPointerSignal: _handleWheelAtBottom, behavior: HitTestBehavior.translucent)`. При положении у конца дня (offset ≥ max - 20) и прокрутке вниз (`PointerScrollEvent`, `scrollDelta.dy > 0`) счётчик увеличивается; после 4 раз вызывается `onDateChanged(следующий день)` и скролл сбрасывается.

4. **Пустой слот: курсор и перетаскивание** — подсказка «Закрепиться за этой точкой» убрана. Для пустого рабочего слота добавлен `MouseRegion(cursor: SystemMouseCursors.grab)` и `GestureDetector(onPanUpdate: ...)`: при «перетаскивании» от пустой ячейки вызывается `_onEmptySlotDragUpdate(delta.dx)` — смещается только горизонтальный скролл колонок мастеров (время и «Нераспределённые» остаются на месте).

5. **Время в миникарточке** — добавлен хелпер `_orderTimeRangeString(order)` (диапазон по `plannedStartTime`/`plannedEndTime` или start + сумма `estimatedMinutes` по позициям). В одной строке с номером заказа сверху отображается «#XXX  10.00–12.30» (и в компактной, и в полной карточке). Дублирующая строка времени в середине полной карточки удалена.

**Как проверить:** Повторный клик по открытой миникарточке закрывает панель. Панель и миникарточка на 1 слот без overflow. У конца дня покрутить колёсико вниз 4 раза — переход на следующий день. На пустом слоте — курсор «рука», перетаскивание двигает только колонки мастеров. В карточках сверху — номер и диапазон времени.

## 2025-03-10 — Overflow панели заказа, Иван-владелец, перелистывание дня, ползунок и клик по слоту

**Дата/время:** 2025-03-10

**Изменённые файлы:**

2. **Иван +79001111111 — владелец**
   - В `AuthNotifier.verifyCode()` при демо-логине по коду: если номер телефона после нормализации равен `79001111111` или оканчивается на `9011111111`, пользователю всегда назначается роль `BusinessRole.owner` и имя «Владелец», независимо от введённого кода.

3. **Перелистывание на следующий день при прокрутке вниз**
   - Добавлен счётчик «лишних» прокруток в конце дня: при `ScrollEndNotification` и позиции скролла внизу (offset ≥ maxScrollExtent - 20) счётчик увеличивается; при достижении 4 вызывается `onDateChanged(selectedDate + 1 день)` и скролл сбрасывается в 0. Счётчик обнуляется, когда пользователь не у конца (offset < max - 30). Вертикальный скролл обёрнут в `NotificationListener<ScrollNotification>`.

4. **Горизонтальный ползунок и закрепление по слоту**
   - Тело сетки мастеров обёрнуто в `Scrollbar(controller: _horizontalBodyScrollController, thumbVisibility: masters.length > 2)`; при более чем двух мастерах ползунок виден. По клику на пустой рабочий слот вызывается `_scrollToMasterColumn(masterIndex)` — горизонтальная прокрутка к колонке этого мастера (`animateTo(masterIndex * _kMasterColWidth)`). У `_SlotCell` добавлен параметр `onEmptySlotTap` и для пустого рабочего слота — `GestureDetector(onTap: onEmptySlotTap)` с подсказкой «Закрепиться за этой точкой (прокрутить к колонке)».

**Как проверить:** Открыть карточку заказа — нет overflow 5px справа. Войти под +79001111111 с любым кодом — роль «Владелец». В расписании прокрутить вниз до конца дня и ещё 4 раза «докрутить» — переключение на следующий день. При 3+ мастерах — виден горизонтальный ползунок; клик по пустому слоту — прокрутка к колонке мастера.
